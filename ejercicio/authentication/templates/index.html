SQL Injection

¿Qué es el SQL Injection?

El SQL Injection (SQLi) es una vulnerabilidad de seguridad común en aplicaciones web que permite a los atacantes ejecutar consultas SQL no deseadas en una base de datos.

Objetivo del ejercicio

El objetivo del ejercicio es modificar el sueldo del usuario con nombre de usuario tomcat. Para ello, tendrá que autenticarse en el sistema como administrador y tendrá que usar ataques de SQL injection para ello.

Cómo prevenir el SQL Injection y el Bypass SQLi:

  • Utilizar consultas parametrizadas o consultas preparadas.
  • Implementar la sanitización de entradas de usuario.
  • Aplicar listas blancas de caracteres para validar la entrada del usuario.
  • Limitar los privilegios de la cuenta de base de datos utilizada por la aplicación.
